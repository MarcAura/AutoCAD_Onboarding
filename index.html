Overview

This guide explains how to set up AutoCAD LT (2019 and 2026), how to install required fonts and plot styles, and how to use eTransmit to share drawings with all dependencies (fonts, CTBs, Xrefs, images, etc.) so files look identical on all systems. It also covers layer management, paper space layouts, a smart title block with attributes, the Block Editor, Sheet Set Manager, and how to produce multi‚Äësheet PDFs quickly.

Layouts (Paper Space)

Layouts represent printable sheets. Model Space = 1:1 design; Layouts = paper with border/title block + scaled viewports.

  1. Switch to a Layout tab.
  2. Insert or Xref your Title Block template.
  3. Create viewport (MV) and set scale (1:2, 1:5, etc.).
  4. Activate viewport, Freeze/Thaw per‚Äësheet layers, then restore the correct Layer State.

Each Layout becomes a page in the final PDF. Use different presets per Layout.

Viewports in AutoCAD

A viewport is a window in a Layout (Paper Space) that looks into Model Space. Use them to control what you see, how big it looks, and which layers are visible.

How to Use Viewports

  1. Switch to a Layout tab (Paper Space).
  2. Type MV (Make Viewport) or use the ribbon to create one.
  3. Draw a rectangle where you want the viewport.
  4. Double-click inside the viewport ‚Üí you‚Äôre now ‚Äúin‚Äù Model Space through that window.
  5. Set the desired scale (e.g. 1:1, 1:2) from the status bar.
  6. Use LAYFRZ or VPFreeze to hide layers in only that viewport.
  7. Double-click outside the viewport to return to Paper Space.

When to Use Viewports

  • To show the same drawing at different scales.
  • To isolate different layers in separate sheets.
  • To make multiple detail callouts from one master model.

‚úÖ Rule of thumb: Always draw in Model Space, always plot from Layouts, and use Viewports to control views.

Fix Gaps, Make a Closed Boundary, then Hatch

Clean the linework ‚Üí make a watertight loop ‚Üí hatch it. Use these tools in order for the least pain.

1) Clean and Join the Linework

  1. Remove duplicates / tiny overlaps
    Run OVERKILL ‚Üí select the edges ‚Üí Enter ‚Üí keep defaults (or enable Optimize segments).
    Removes stacked/overlapping bits that break hatches.
  2. Join into one polyline (with a fuzz)
    PEDIT ‚Üí M (Multiple) ‚Üí select all lines/arcs ‚Üí Enter ‚Üí J (Join) ‚Üí when asked for fuzz distance, enter something tiny (e.g., 0.01 inches or 0.25 mm) ‚Üí Enter ‚Üí C (Close) if needed.
    Connects endpoints within that tolerance and merges arcs/lines into a single closed polyline.
  3. Snap open corners shut (if needed)
    FILLET ‚Üí R ‚Üí radius 0 ‚Üí click the two edges at each open corner.
    Zero-radius fillet seals corners perfectly. Use TRIM/EXTEND if things don‚Äôt meet, then re-PEDIT ‚Üí J/C.

Turn on OSNAP (F3) and ensure Endpoint/Intersection are active to avoid creating new gaps.

2) (Alternate) Auto-trace a Boundary

If joining is messy: BOUNDARY ‚Üí Object type: Polyline ‚Üí Pick Points inside the area ‚Üí Enter.
This creates a fresh, watertight polyline to hatch.

3) Hatch It (with Gap Help if Needed)

Two reliable ways

A) Hatch from the closed polyline (most robust)

  1. HATCH ‚Üí Select boundary objects ‚Üí click your closed polyline ‚Üí Enter.

B) Hatch by pick-point with gap tolerance

  1. Set a small gap tolerance: HPGAPTOL = e.g., 0.01 (in) or 0.25 (mm).
  2. (Optional) Show gap markers: HPGAPCOLOR = a bright index (e.g., 1 = red).
  3. HATCH ‚Üí Pick internal point ‚Üí click inside ‚Üí Enter. If it fails, bump HPGAPTOL slightly and retry.

4) Make the Hatch Behave Nicely Later

  • Associativity: set HPASSOC = 1 so the hatch updates when you edit the boundary.
  • Layering: put it on a hatch layer (or preset HPLAYER).
  • Edit later: use HATCHEDIT to tweak scale/pattern/associativity.

Quick Command Cheat-Sheet

PEDIT (M‚ÜíJ)   Join edges with fuzz; C to close
FILLET R=0    Snap open corners together
BOUNDARY      Auto-trace a clean polyline

Solid Hatch Shows Blank Areas (Even on Plot)

Fixes for tiny 0.004" traces, dense vertices, and ‚Äúignored‚Äù corners when the boundary is fine but the hatch doesn‚Äôt fully fill.

1) Use a Clean, Simple Boundary

  1. OVERKILL the boundary sources (removes dups/overlaps).
  2. PEDIT ‚Üí M (Multiple) ‚Üí select segments ‚Üí J (Join) ‚Üí give a tiny fuzz (e.g., 0.001‚Äì0.01 in) ‚Üí C (Close) if needed.
  3. If still fussy: BOUNDARY ‚Üí Object type: Polyline ‚Üí Pick Points inside ‚Üí Enter (use this fresh polyline as your hatch boundary).

2) Make the Hatch Ignore Micro-Islands

  • Set HPISLANDDETECTION = 2 (Ignore islands).
    Prevents tiny sliver ‚Äúholes‚Äù created by ultra-thin traces from punching through the SOLID fill.
  • (Ribbon) Hatch Creation ‚Üí Options ‚Üí Island detection: Ignore.

3) Raise Hatch Complexity Limits

  • Set HPMAXAREAS high (e.g., 25000).
    Allows more separate hatch sub-areas before AutoCAD stops processing them.
  • (Pattern hatches only) If you later use line-based patterns, also bump HPMAXLINES (e.g., 2000000).

4) Ensure Fills Actually Plot/Display

  • FILLMODE = 1 ‚Üí REGENALL.
    Global switch for showing/plotting fills and solids.
  • Hatch Transparency = 0 (or make sure ‚ÄúPlot transparency‚Äù is enabled when plotting).

5) Recreate the Hatch the Robust Way

  1. Delete the glitchy hatch (keep the polyline boundary).
  2. HATCH ‚Üí set Island detection to Ignore ‚Üí Select boundary objects ‚Üí click the single closed polyline ‚Üí Enter.
  3. Set HPASSOC = 1 so it stays linked to the boundary.

6) Fix Draw-Order Conflicts

  • Select hatch ‚Üí Draw Order ‚Üí Bring to Front (or send under linework if desired).
  • Watch for images/wipeouts/masks above the hatch‚Äîthose can create apparent blank spots.

7) Last-Resort Geometry Tricks (for ultra-thin 0.004" detail)

  • OFFSET the boundary outward by a hair (e.g., 0.001‚Äì0.002 in) to remove micro-slivers, then hatch that copy.
  • Convert boundary to REGION ‚Üí fix self-intersections with UNION/SUBTRACT ‚Üí hatch the region.

Quick Vars/Commands

HPISLANDDETECTION = 2   (Ignore)
HPMAXAREAS = 25000      (or higher)
FILLMODE = 1            (then REGENALL)

[Draw Order] Bring to Front / Send to Back

HATCH HEALING & FIXING BLANK AREAS

Purpose

If your hatch fills only part of a closed polyline, or leaves blank spots even though the boundary looks fine, use these steps to rebuild or clean the boundary so hatching works correctly.

Method 1: Rebuild Boundary as REGION

  1. Select closed polyline ‚Üí run REGION (creates watertight boundary).
  2. If it errors, fix intersections: PEDIT / FILLET R=0 / TRIM as needed, then retry.
  3. Re-hatch: HATCH ‚Üí Select boundary objects ‚Üí choose REGION ‚Üí SOLID.

Method 2: Force a Chunkier Boundary

  1. Duplicate the boundary.
  2. OFFSET outward by 0.001 in (or inward if gaps are internal).
  3. Hatch the offset copy ‚Üí erase the original if needed.
  4. This helps eliminate tiny islands that confuse hatching.

Method 3: Simplify Extreme Vertex Counts

  1. Run OVERKILL (check ‚ÄúCombine co-linear‚Äù).
  2. Then use PEDIT ‚Üí Straighten (S) or Fit/Spline.
  3. Re-hatch ‚Üí fewer noisy vertices = fewer problems.

Method 4: Plot as Raster

  1. Plot with DWG To PDF.pc3.
  2. In Plot dialog ‚Üí Printer/Plotter ‚Üí Properties ‚Üí bump raster resolution ‚Üí enable Plot Transparency.
  3. This forces raster output for hatch ‚Üí often removes ‚Äúholes‚Äù in PDF/print.

Method 5: Draw Order + Overlay Checks

  • Select hatch ‚Üí Draw Order ‚Üí Bring to Front (or Send Under if needed).
  • Temporarily turn off images, wipeouts, or masks to see if they cause gaps.

Use these when a hatch leaves unexpected empty areas even though the boundary looks closed. FYI: In your case, OVERKILL resolved it‚Äîgreat call.

1) Rebuild the boundary as a REGION, then re‚Äëhatch

  1. Select your closed polyline ‚Üí run REGION (creates a mathematically ‚Äúwatertight‚Äù boundary).
  2. If REGION errors, you likely have a self‚Äëintersection. Fix with PEDIT / FILLET R=0 / TRIM, then try REGION again.
  3. Hatch the REGION: HATCH ‚Üí Select boundary objects ‚Üí pick the region ‚Üí choose SOLID.

2) Force a ‚Äúchunkier‚Äù boundary to eliminate micro‚Äëslivers

  2. OFFSET outward by a hair (e.g., 0.001 in) ‚Üí hatch the offset copy ‚Üí then erase the original.
  3. If the missing areas are internal, try offsetting inward instead.
  4. This often bulldozes tiny islands the solver interprets as holes.

3) Simplify extreme vertex counts

Appendix C ‚Äî Worked Example (Existing + New)

Part 1: Organizing an Existing Design

Goal: Reorganize geometry, apply Layer States, build Layouts, insert Title Block with attributes, publish multi‚Äëpage PDF.

  1. Identify Geometry Categories ‚Äî TRACES, FIDUCIALS, ADHESIVE, NOTES.
  2. Create Standard Layers ‚Äî LAYER:

    • Layout ‚ÄúAdhesive‚Äù
    • Layout ‚ÄúFinal Assembly‚Äù
  6. Insert Title Block Attributes ‚Äî REV/PROJECT/DRAWN_BY/DATE ‚Üí ATTSYNC; set REV = A.

Part 3: Benefits of This Workflow
